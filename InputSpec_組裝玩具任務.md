## 1. 關鍵發現:現在的切換是「三段式模態」,不是 GDD 設想的「直接切焦點」

**現狀流程**(`handleSelection` :154):
```
[選擇模式] 方向鍵移動游標選格子
   → 按 A「激活」該零件 → [操作模式] 此後所有輸入歸該零件用
   → 按 B「退出」回選擇模式 → 再用方向鍵選 → 再按 A …
```
一次零件切換 = **B → 方向鍵(可能數下)→ A**,至少 3 個按鍵動作。而 GDD §4 的設想是「方向鍵按下去焦點就換、選中即可操作」= **1 個動作**。

**這就是『卡卡』最可能的結構性根因**:不是程式跑得慢,而是**操作模型本身讓切換又貴又慢**。在「手忙腳亂、緊張爽快」的目標下,每次切換多出 2+ 個按鍵,忙亂會變成煩躁。
→ 對 P3 的意義:P3 不只是「修效能」,更是**把模態切換改成直接切換**的操作模型重構。

**次要發現**:激活狀態下方向鍵被零件佔用(如 CLAW 用上下鍵轉臂),所以「不退出就不能切」是必然結果;另外游標初始為空(:179),開場第一下方向鍵只是「初始化游標」,沒有實際移動——也會讓人覺得「按了沒反應」。

## 3. 預期手感(P3/P4/A2 的驗收基準)

1. **切換零延遲**:上/下鍵按下的**當幀**焦點就換、高亮就跳、crank/左右鍵/A 立即接管新零件。不允許任何確認步驟或過場。
2. **crank 不留殘值**:切換焦點後,crank 的轉動只影響新焦點;前一個零件的角度停在原處(狀態保留,見 P4)。
3. **移動的按壓感**(A2 參數):左右鍵移動的加速/停止手感可調,避免呆板。
4. **開場即可操作**:任務開始時預設焦點=移動零件(下排),不需要先按一下初始化游標。
5. **CLAW 抓/放明確**(D4 新制):A 按下當幀執行抓(範圍內有物)或放;爪子開合為自動演出動畫,玩家不需自己控制開合角。
